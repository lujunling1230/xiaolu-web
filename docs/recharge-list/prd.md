## 1. 产品背景与定位

### 1.1 产品愿景

在快节奏的现代生活中，情绪疲劳已成为一种普遍现象。用户普遍面临注意力碎片化、能量耗散感强、自我关怀不足的问题。回血清单旨在为用户提供一个轻量级的"微型能量站"，通过清单化的方式引导用户完成一件件滋养身心的小事，帮助用户在日常中完成情绪修复与能量恢复。

### 1.2 产品定位

回血清单是妙妙工具箱「心」维度下的核心工具，与系统调频功能共同组成情绪关怀模块。其核心隐喻是"给生命充电"：每一件小事是一个能量节点，用户点击完成即相当于接通电源，获得能量补充。

### 1.3 核心价值主张

- **降低行动门槛**：将"照顾自己"这一抽象目标拆解为100+具体可执行的微小行动，用清单格式降低决策成本。
- **即时正反馈**：每次完成动作伴随视觉（充能粒子爆发）、听觉（电流声）、数据（计数值）三重反馈，强化成就感。
- **哲学收纳**：通过第101个隐藏电源"允许自己永远充不满电"，消解用户的完成焦虑，传递自我接纳的价值观。
- **周维度视角**：以周为单位统计回血次数，帮助用户建立可持续的自我关怀节奏。

|----------|----------|--------|----------|

| M-02 | 胶囊电池组件 | P0 | 每件小事的独立UI组件，包含图标、文字、电池动画 |
| M-03 | 本周回血计数 | P0 | 毛玻璃固定右下角，记录本周完成次数，周自动重置 |
| M-04 | 第101个隐藏电源 | P1 | 底部隐藏按钮，点击弹出哲学弹窗+小猫动画联动 |
| M-05 | 小猫SVG动画 | P1 | 底部装饰小猫，睡眠/睁眼/哈欠三态切换，与隐藏电源联动 |
| M-06 | 心电图背景纹理 | P1 | 页面加载时波动动画，营造"能量检测"氛围 |
| M-07 | 数据持久化 | P0 | localStorage存储完成状态与周计数，支持页面刷新恢复 |

### 3.2 模块详情

#### M-01 能量节点流

**功能描述**：以垂直列表形式展示100件回血小事，每件小事是一个独立可点击的能量节点。用户浏览时节点以交错淡入+左滑动画入场（`whileInView`触发）。

**数据来源**：硬编码100条中文任务文案，配以100个图标池（emoji + 中文字符），按索引循环分配。

**交互规则**：
- 点击未完成节点：标记为已完成，触发充能粒子爆发动画（8个绿点向外扩散），播放电流声（Web Audio API，sawtooth波形，频率800Hz->400Hz，持续0.15s），本周计数+1
- 点击已完成节点：取消完成状态，还原至未完成
- 节点间以连接线（带流动小点）串联，完成节点连接线变为绿色

**视觉状态**：
- 默认态：暖白半透明胶囊背景，图标灰度40%，文字灰色
- 悬停态：边框变金，背景亮度提升，阴影增强
- 完成态：边框变绿，背景浅绿，图标恢复彩色，文字变绿，电池填满

#### M-02 胶囊电池组件

**功能描述**：每个能量节点由三部分组成：左侧图标、中间文字、右侧电池指示器。

**电池动画**：
- 未完成：电池为空心，填充宽度为0
- 完成：电池填充宽度从0过渡到100%（cubic-bezier缓动），绿色渐变填充，外发光效果

**连接线**：
- 节点间有2px宽的垂直连接线
- 连接线上有3px圆点，hover时金黄色流动，完成时绿色流动

#### M-03 本周回血计数

**功能描述**：页面右下角固定悬浮的毛玻璃卡片，显示"本周已回血 N 次"。

**计数规则**：
- 每次点击完成（从未完成到完成）时计数+1
- 取消完成不减少计数
- 每周一00:00自动重置为0
- 重置依据：`recharge_count_ts`记录的时间戳，对比本周一零时

**动画**：计数变化时组件以spring动画缩放弹入（`scale: 0.9 -> 1`）

**样式**：毛玻璃效果（`backdrop-filter: blur(12px)`），半透明白底，圆角胶囊

#### M-04 第101个隐藏电源

**功能描述**：列表底部一个极小的圆形按钮（6px），几乎不可见，点击后弹出全屏遮罩弹窗。

**弹窗内容**：
- 标题："第101个电源"（Noto Serif SC，24px，白色）
- 正文："允许自己永远充不满电。"（14px，半透明白色）
- 装饰分割线
- 关闭按钮（右上角X）

**弹窗样式**：毛玻璃卡片（`backdrop-filter: blur(20px)`），半透明白底，暗色遮罩

**联动逻辑**：首次点击隐藏电源时，小猫从"睡眠"状态切换到"睁眼"（2s后）-> "哈欠"（3.5s后）-> "睡眠"循环

#### M-05 小猫SVG动画

**功能描述**：底部装饰性SVG小猫，三种状态循环或联动。

**三种状态**：
| 状态 | 触发条件 | 视觉表现 |
|------|----------|----------|
| sleep | 默认状态 | 闭眼弧线，Zzz文字，呼吸上下浮动动画 |
| stare | 点击隐藏电源后2s | 大圆睁眼，蓝色瞳孔+高光，头部轻微抖动 |
| yawn | stare后1.5s | 闭眼弧线，张大嘴（红色椭圆），哈欠表情 |

**小猫结构**：SVG椭圆身体+圆形头部+三角耳朵+尾巴+胡须+鼻子+嘴

#### M-06 心电图背景纹理

**功能描述**：页面背景使用SVG data URI作为重复纹理，模拟心电图波形。

**技术实现**：SVG图案编码为data URI，以`background-image`重复平铺（400x120px单元）。

**加载动画**：页面挂载时，通过CSS伪元素叠加一层相同纹理，以`ecg-flash`关键帧动画模拟"心电图跳动"效果（1.2s，透明度0->0.4->0）。

#### M-07 数据持久化

**存储键值**：

| 键名 | 类型 | 说明 |
|------|------|------|
| `recharge_count` | string(Number) | 本周完成计数 |
| `recharge_done_ids` | string(JSON Array) | 已完成节点ID数组 |
| `recharge_count_ts` | string(Number) | 最近一次计数操作的时间戳 |

**周重置逻辑**：
1. 读取`recharge_count_ts`时间戳
2. 计算本周一00:00:00的Date对象
3. 若时间戳早于本周一，则视为新的一周，计数归零
4. 归零时不清除`recharge_done_ids`中的完成状态（用户可保留历史完成记录）

**容错**：所有localStorage读写均包裹`try-catch`，失败时静默降级。

## 4. 数据模型

### 4.1 能量节点

```
interface EnergyNode {
  id: number;        // 1-100，唯一标识
  text: string;      // 事项描述文案
  icon: string;      // 分配的图标字符
}
```

### 4.2 完成状态

```
// 已完成节点ID集合（localStorage存储）
type DoneIds = Set<number>;  // 序列化为 JSON Array

// 周计数
type WeekCount = number;     // 当前周累计完成次数

// 重置时间戳
type CountTimestamp = number; // Date.now() 毫秒级时间戳
```

### 4.3 小猫状态机

```
State: "sleep" | "stare" | "yawn"
Transitions:
  sleep -> stare  (触发条件: 首次点击隐藏电源)
  stare -> yawn   (触发条件: stare持续2s后自动)
  yawn  -> sleep  (触发条件: yawn持续1.5s后自动)
```

## 5. 交互逻辑与规则约束

### 5.1 状态流转规则

| 当前状态 | 用户操作 | 新状态 | 条件 |
|----------|----------|--------|------|
| 未完成 | 点击节点 | 已完成 | 总是允许 |
| 已完成 | 点击节点 | 未完成 | 总是允许（允许反悔） |
| 任意 | 点击隐藏电源 | 弹窗打开 | 总是允许 |
| 弹窗打开 | 点击关闭/遮罩 | 弹窗关闭 | 总是允许 |

### 5.2 计数规则

- 同一节点在同一周期内多次点击（完成->取消->完成），每次"完成"操作均计入计数+1
- 取消完成操作不计入计数（也不减少计数）
- 页面刷新后，已完成节点的再次点击，即使计数已重置，仍可切换状态

### 5.3 性能约束

- 节点列表无虚拟滚动（100个节点总量可控）
- 粒子动画使用CSS `motion.div`，8个粒子/次，动画时长600ms
- Web Audio上下文延迟创建（首次点击时实例化），避免自动播放策略限制
- SVG背景纹理使用encodeURIComponent编码，避免额外HTTP请求

### 5.4 浏览器兼容性

- 依赖`backdrop-filter`属性（毛玻璃效果），不支持时降级为纯色半透明背景
- 依赖`framer-motion`动画库，需确保`AnimatePresence`正常工作
- Web Audio API需在用户手势后初始化（遵循浏览器自动播放策略）

## 6. 验收标准

### 6.1 功能验收

| 编号 | 验收项 | 验收标准 | 优先级 |
|------|--------|----------|--------|
| AC-01 | 节点列表渲染 | 100个节点按序渲染，每个包含图标、文字、电池 | P0 |
| AC-02 | 点击完成反馈 | 点击节点后：电池填满、粒子爆发、电流声播放、计数+1 | P0 |
| AC-03 | 完成状态持久化 | 刷新页面后已完成节点保持完成态 | P0 |

| AC-06 | 隐藏电源弹窗 | 点击底部小圆点，弹出毛玻璃弹窗，显示哲学文案 | P1 |

| AC-08 | 心电图加载动画 | 页面首次加载时，背景纹理出现闪动动画 | P1 |

## 7. 未来迭代方向

### 7.1 V1.1 规划

- **自定义小事**：允许用户添加自己的回血小事，补充到列表中
- **分类筛选**：按场景（户外、居家、社交、独处）筛选节点
- **分享卡片**：生成本周回血次数的分享图
- **音效开关**：提供全局音频开关，允许用户关闭电流声

### 7.2 V1.2 规划

- **每日推荐**：每天随机推荐3-5件小事，降低选择困难
- **连续回血**：记录连续回血天数，增加连续奖励视觉
- **回血日记**：完成节点后可选填写一句话感受
- **数据导出**：支持导出回血记录为JSON/CSV

### 7.3 V2.0 规划

- **社交回血**：好友间互相推荐回血小事，查看好友本周回血次数
- **AI回血官**：基于用户历史偏好，AI生成个性化回血建议
- **回血播客**：内嵌白噪音/冥想音频播放器
- **多端同步**：通过账号体系实现跨设备数据同步
