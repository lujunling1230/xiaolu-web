## 一、产品背景

原"系统调频"是基于李松蔚《5%的改变》的认知轨解忧工具，以"电台调频"为隐喻。

现将其升级为**"爱情公寓·元宇宙客厅"**——借用经典情景喜剧《爱情公寓》的人物设定，将 7 位性格迥异的角色转化为 AI 解忧伙伴，让用户像串门一样在客厅中随意呼叫任意房客，插科打诨中解决人生 Bug。

---

## 二、产品定位

> 一个以群居公寓为场景、全员 AI 驱动的互动式情绪与认知支持系统。
>
> 用户是刚搬入 3603 的"第 8 位神秘房客"，可随时连线 3601/3602 的 7 位老友。

## 四、页面结构

### 1. 客厅大厅 `/toolbox/answer`

- 顶部："🏠 3601 · 3602 · 3603" 门牌 + "爱情公寓·元宇宙客厅" 标题
- 副标题："你住在 3603，产品经理。点击任意房客，开始插科打诨、解决人生 Bug。"
- 7 位角色以卡片网格排列（auto-fill，桌面 3-4 列，手机 2 列）
- 每张卡片：emoji 头像 + 姓名 + 称号 + 房间号 + 频率号
- 底部提示：群聊触发关键词引导

### 2. 单聊界面

- 头部：角色 emoji + 电台标题 + 房间号 + 频率 + 在线状态
- 消息区：用户气泡（右，深色）/ 角色气泡（左，角色主题色）
- 空状态：显示角色口头禅 + 输入引导
- 输入框：多行文本 + 发送按钮（角色主题色）
- Enter 发送，Shift+Enter 换行

### 3. 群聊模式（彩蛋）

- 触发条件：用户输入包含特定关键词
- 触发后：2-3 位角色依次接力回复
- 每条回复间隔 600ms，带"正在输入"动画
- 群聊头部显示"群聊模式 · 多人接力吐槽中…"

## 六、技术实现

### 6.1 AI 调用

- 复用 `src/utils/aiClient.ts`（`callAI` 函数）
- 模型：`deepseek-r1`（可覆盖）
- 每个角色独立 System Prompt，通过 `src/data/characters.ts` 配置

### 6.2 群聊上下文

- 单聊：仅当前角色的 System Prompt + 用户输入
- 群聊：角色 System Prompt + 前 2 条群聊回复作为上下文 + 用户输入

### 6.3 Web Audio 音效

| 场景 | 音效 |
|------|------|
| 点击角色卡片 | 拨号音（3 声"嘟"） |
| 进入聊天 | 接通音（清脆"叮"） |
| 收到回复 | 同上 |
| 触发群聊 | 轻快和弦（3 音阶） |
