## 1. 产品背景与定位

### 1.1 产品愿景

现代人的压力来源多元且持续，从工作焦虑到社交倦怠，从信息过载到决策疲劳。传统的解压方式（运动、冥想、社交）往往需要较高的启动成本。解压馆旨在提供一种"零门槛即点即玩"的触觉发泄体验，让用户在任何时刻都能找到一种简单、即时、无负担的释放方式。

### 1.2 产品定位

解压馆是妙妙工具箱「感」维度下的核心工具，聚焦于感官层面的即时解压。通过三款马卡龙风格的小游戏，分别对应触觉（捏泡泡）、视觉（切割方块）、动觉（重力涂鸦）三种感官通道，为用户提供丰富多样的解压选择。

### 1.3 核心价值主张

- **零学习成本**：即点即玩，无需教程，直觉化交互
- **多感官通道**：捏泡泡（触觉+听觉）、禅意切割（视觉+动觉）、重力涂鸦（动觉+视觉），覆盖不同解压偏好
- **无限循环**：所有游戏内容可无限重生，无结束条件，无压力循环
- **美学治愈**：马卡龙色系、圆角卡片、柔和渐变，视觉本身即具疗愈效果

### 3.2 模块详情

#### M-01 游戏选择卡片

**功能描述**：首页展示三张横向卡片，每张代表一个游戏。卡片包含图标（渐变背景圆形）、游戏名称、一句话描述、"开始"按钮。

**卡片数据**：

| 游戏 | 图标 | 描述 | 渐变背景 |
|------|------|------|----------|
| 无限捏泡泡 | 泡泡 | 永远捏不完的满足。 | 粉->蓝渐变 |
| 禅意切割 | 刀 | 一刀两半，万物可裂。 | 绿->紫渐变 |
| 重力涂鸦 | 铅笔 | 画下的都会落下。 | 黄->粉渐变 |

**交互**：
- 点击卡片进入对应游戏（带缩放过渡动画，`scale: 0.96 -> 1`）
- hover时卡片上浮6px，阴影增强，边框变粉色
- 桌面端卡片横向排列，移动端（640px以下）纵向堆叠

**过渡动画**：使用`AnimatePresence mode="wait"`确保卡片退出和游戏进入之间有平滑过渡

#### M-02 无限捏泡泡 (Pop It)

**功能描述**：6x6 SVG气泡网格，共36个彩色圆形气泡。点击任意气泡触发爆裂效果。

**技术实现**：
- SVG容器：`(6 * (44 + 14) + 14) = 362px` 正方形
- 每个气泡半径：`CELL/2 - 2 = 20px`
- 颜色分配：按`(row * COLS + col) % 6`取马卡龙色板

**爆裂动画**：
- 气泡本身：`scale: 1 -> 0`, `opacity: 1 -> 0`，持续250ms
- 星星粒子：6个"✦"字符，向随机方向飞散，伴随旋转180度，持续600ms
- 音效：Web Audio播放"啵"声（sine波形，720Hz->220Hz，持续100ms）

**重生逻辑**：
- 爆裂后600ms，气泡在相同位置重生
- 重生时随机分配新颜色（从马卡龙色板中随机选取）
- 星星粒子在700ms后从DOM中清除

**规则约束**：
- 已爆裂的气泡不可再次点击（`popped === true`时`onClick`被阻止）
- 无游戏结束条件，无分数统计

#### M-03 禅意切割 (Zen Cut)

**功能描述**：在5个彩色方块组成的画布上，用户通过拖拽划线，穿过方块时将其切割成两半，上下两半分别向相反方向飞散旋转消失，随后方块在随机位置重生。

**技术实现**：
- 方块尺寸：90-110px宽 x 70px高，圆角16px
- 初始位置：交错排列，轻微偏移
- 交互：PointerEvent（兼容触控和鼠标），在`stageRef`上监听`onPointerDown/Move/Up/Leave`
- 切割判定：收集拖拽轨迹中落入方块范围内的点，取首尾两点作为切割线
- 切割实现：`clipPath: polygon()`将方块分为上下两部分

**飞散动画**：
- 上半部分：向左上飞散（`x: -6, y: -50, rotate: -14, opacity: 0`），持续700ms
- 下半部分：向右下飞散（`x: 8, y: 80, rotate: 18, opacity: 0`），持续700ms
- 动画完成后（`onAnimationComplete`）触发重生

**重生逻辑**：
- 在画布范围内随机位置（x: 40-300, y: 50-230）
- 随机分配新颜色
- 切割状态重置为null

**视觉细节**：
- 未切割方块有轻微上下浮动动画（`y: [0, -4, 0]`），不同方块有不同延迟
- 方块有内阴影高光，模拟立体感
- 切割轨迹以白色折线实时显示
- 每次切割使用唯一`cutKey`确保React正确识别新切割

#### M-04 重力涂鸦 (Gravity Doodle)

**功能描述**：用户在Canvas上画线，实时生成彩色圆点。圆点受重力影响向下坠落，落到已有堆积物上或画布底部时停止，形成彩色堆积景观。

**技术实现**：
- Canvas渲染，使用`devicePixelRatio`适配高清屏
- `requestAnimationFrame`循环驱动物理模拟（60fps）
- 圆点属性：`{x, y, vx, vy, r, color, settled}`

**物理模型**：
- 重力加速度：`0.35` 像素/帧
- 圆点半径：`4 + Math.random() * 3`（4-7px随机）
- 水平速度初始为0，无水平阻力

**堆积检测**：
- 高度图数组：按8px列宽划分画布，每列记录当前最高堆积点的Y坐标
- 圆点每帧检测：`d.y + d.r >= heightMap[col]` 时停止
- 停止后更新该列及相邻列的高度图，形成堆积斜面效果
- 越界回收：`x < -20 或 x > w + 20` 时标记为settled

**插值生成**：
- 拖拽时在两点间插值生成连续圆点，间隔4px
- 确保画线流畅无断点

**性能控制**：
- 圆点总量限制：超过600个时清除最早的100个
- 防止Canvas绘制过载导致帧率下降

**清空功能**：
- 右下角悬浮清空按钮（垃圾桶图标）
- 点击后清空所有圆点，重置高度图
- hover时按钮放大1.1倍，背景变粉

#### M-05 游戏切换

**功能描述**：游戏内左上角显示"换一个"按钮，点击后返回卡片选择页。

**过渡动画**：
- 游戏视图退出：`opacity: 0, scale: 0.96`
- 卡片列表进入：`opacity: 1`
- 使用`AnimatePresence mode="wait"`确保退出动画完成后再渲染卡片

**定义**：

## 4. 数据模型

### 4.1 气泡（Pop It）

```
interface Bubble {
  id: number;        // row * COLS + col
  cx: number;        // 圆心X
  cy: number;        // 圆心Y
  r: number;         // 半径
  color: string;     // 马卡龙色值
  popped: boolean;   // 是否已爆裂
}
```

### 4.2 星星粒子

```
interface Star {
  id: number;      // 自增ID
  x: number;       // 起始X
  y: number;       // 起始Y
  dx: number;      // 终点偏移X
  dy: number;      // 终点偏移Y
  color: string;   // 随机马卡龙颜色
}
```

### 4.3 方块（Zen Cut）

```
interface Block {
  id: number;       // 0-4
  x: number;        // 左上角X
  y: number;        // 左上角Y
  w: number;        // 宽度
  h: number;        // 高度
  color: string;    // 马卡龙色值
  cut: {            // 切割信息，null表示未切割
    rx1: number;    // 相对x1
    ry1: number;    // 相对y1
    rx2: number;    // 相对x2
    ry2: number;    // 相对y2
    key: number;    // 唯一键（用于React识别）
  } | null;
}
```

### 4.4 重力圆点（Gravity Doodle）

```
interface GDot {
  x: number;       // 当前位置X
  y: number;       // 当前位置Y
  vx: number;      // 水平速度
  vy: number;      // 垂直速度
  r: number;       // 半径
  color: string;   // 马卡龙色值
  settled: boolean; // 是否已停止
}
```

## 5. 交互逻辑与规则约束

### 5.1 游戏状态机

```
[卡片选择页] --点击卡片--> [游戏进行中] --点击"换一个"--> [卡片选择页]
```

### 5.2 捏泡泡规则

- 已爆裂的气泡（`popped === true`）不响应点击
- 爆裂后600ms自动重生，380ms后星星粒子清理
- 无最大点击次数限制，可无限循环

### 5.3 禅意切割规则

- 切割需要拖拽轨迹中至少有2个点落入方块内
- 已切割的方块在动画完成前不可再次切割（`b.cut`存在时跳过）
- 一个方块被切割后，约700ms后重生
- 允许多个方块同时被切割（如果轨迹穿过多个方块）

### 5.4 重力涂鸦规则

- 圆点总量限制600个，超出时FIFO删除最早100个
- 画线在两点间插值生成（间隔4px），保证连续
- 圆点初始无水平速度，仅受重力影响
- 高度图按8px列宽检测，相邻列更新形成堆积斜面

### 5.5 性能约束

- Pop It：36个气泡+最多36个星星粒子，SVG渲染，性能开销低
- Zen Cut：5个方块，最多10个切割碎片，DOM渲染，性能开销低
- Gravity Doodle：最多600个圆点，Canvas渲染，60fps目标
- 所有游戏切换使用`AnimatePresence`确保DOM节点正确清理

### 5.6 浏览器兼容性

- Canvas API需基础支持
- PointerEvent需浏览器支持（Chrome 55+, Safari 13+, Firefox 59+）
- Web Audio API需在用户手势后初始化
- `clipPath` CSS属性需浏览器支持（Chrome 55+, Safari 9.1+, Firefox 54+）

## 7. 未来迭代方向

### 7.1 V1.1 规划

- **捏泡泡模式切换**：支持切换圆点/心形/星形气泡形状
- **捏泡泡震动反馈**：移动端调用Vibration API，捏泡泡时触发触觉震动
- **禅意切割音效**：切割时增加"撕裂"音效，增强沉浸感
- **重力涂鸦色彩选择**：提供调色板，允许用户选择单色或多色模式

### 7.2 V1.2 规划

- **新游戏：揉纸团**：模拟揉纸团的声音和触感，纸团可丢入垃圾桶
- **新游戏：撕纸**：模拟撕纸的触感，支持不同纸张纹理
- **重力涂鸦：橡皮擦**：支持局部擦除圆点
- **捏泡泡：连击计数**：记录连续戳破气泡的速度和数量

### 7.3 V2.0 规划

- **解压统计**：记录用户对各游戏的使用时长和频率
- **个性化推荐**：基于使用习惯推荐解压游戏
- **AR解压**：利用WebXR实现AR捏泡泡/切割体验
- **社交模式**：支持好友间互相发送"解压礼物"
