## 一、产品背景与定位

### 1.1 产品背景

时光博物馆是妙妙工具箱（Toolbox）中「魂」维度的核心产品，承载着个人成长历程的数字化叙事。在工具箱的四大维度（器、术、法、魂）中，「魂」维度聚焦于自我认知与内在沉淀，而时光博物馆作为该维度的压轴篇章，旨在将个人经历、荣誉与时代记忆融合为一场可回溯、可交互的沉浸式展览。

本产品以"博物馆"为隐喻，将用户的人生片段划分为四个主题展厅，通过复古胶片美学与粒子动画营造时光沉淀的沉浸感。每一件展品（卡片）都承载着特定年份的记忆，既是个人档案的数字化呈现，也是时代背景的缩影。

### 1.2 产品定位

| 维度 | 描述 |
|------|------|
| **所属模块** | 妙妙工具箱 / 魂维度 |
| **路由** | `/toolbox/memories` |
| **页面组件** | `src/pages/toolbox/MuseumPage.tsx` |
| **产品形态** | 单页 Web 应用，四个展厅纵向排列 |
| **核心价值** | 个人成长叙事 + 时代记忆存档 + 自我复盘反思 |
| **目标用户** | 个人站长（管理员）及其访客 |

### 1.3 产品目标

- 提供一套完整的个人荣誉与时代记忆展示系统
- 通过复古胶片设计语言建立独特的情感共鸣
- 支持管理员灵活增删改查展品内容
- 以 localStorage 实现零后端依赖的数据持久化

## 二、用户画像与核心场景

### 2.1 用户画像

**管理员（Owner）**
- 身份：个人站长，即工具箱所有者
- 行为：维护与更新展品内容，添加个人荣誉、时代记忆条目
- 需求：希望有一处集中的空间展示个人成长轨迹与时代印记，管理操作简单直观
- 权限：通过密码校验后可执行 CRUD 操作

**访客（Visitor）**
- 身份：任意访问工具箱的外部用户
- 行为：浏览四个展厅，查看展品详情，播放音乐，展开复盘内容
- 需求：沉浸式体验个人成长故事，了解时代文化背景
- 权限：仅浏览，无编辑权限

### 2.2 核心场景

**场景一：展示个人荣誉**
管理员登录后，进入「荣耀之路」展厅，添加一条新的荣誉记录（如"获得某某竞赛一等奖"），上传证书图片，填写年份、描述与复盘反思。访客浏览时可展开查看复盘内容，了解荣誉背后的成长故事。

**场景二：构建时代记忆库**
管理员在「时代回响」展厅的三个子类目中，添加具有年代代表性的歌曲、影视作品、网络文化条目。例如添加一首 2005 年的经典歌曲，填入网易云音乐外链，访客可点击播放并唤起怀旧情感。

**场景三：日常维护与更新**
管理员发现某条展品信息有误，进入编辑模式，修改描述或替换图片。删除过时内容时，系统弹出确认对话框防止误操作。

**场景四：数据迁移与兼容**
系统升级后，旧数据中缺少新增的 `imageUrl` 字段。页面加载时自动检测并补全，确保历史数据无缝兼容。

### 3.2 展厅一：时代回响（Hall of Echoes）

#### 3.2.1 子类目结构

该展厅包含三个子类目，通过顶部 Tab 切换：

| 子类目 | Tab 标签 | 默认数据量 | 数据结构 Key |
|--------|----------|------------|-------------|
| 时代 BGM | 时代BGM | 12 首 | `museum_bgms` |
| 时代荧幕 | 时代荧幕 | 11 部 | `museum_tvs` |
| 时代网络 | 时代网络 | 3 条 | `museum_nets` |

#### 3.2.2 卡片字段定义

每张卡片包含以下字段：

- **年份（year）**：字符串，标识该展品所属年代
- **标题（title）**：字符串，展品名称
- **描述（description）**：字符串，展品简介
- **封面图（imageUrl）**：字符串，支持 Base64 编码或外部 URL
- **音乐链接（musicUrl）**：字符串，仅 BGM 类目，网易云音乐 iframe 外链

#### 3.2.3 交互规则

- Tab 切换时，使用 Framer Motion 动画过渡展示对应类目的卡片列表
- 卡片以复古报纸风格呈现：深褐底色（`#3a2a1a` 附近），暗金边框（`#b08d57`），衬线字体
- BGM 类目的卡片附带迷你播放器按钮，点击后弹出网易云音乐 iframe 播放器
- 管理员模式下，每张卡片右上角显示编辑和删除操作按钮
- 管理员模式下，列表底部显示"添加展品"按钮，点击弹出表单

#### 3.2.4 默认数据内容概要

**时代BGM（12首）**：涵盖 2000-2015 年间的经典华语及流行歌曲，每首附带网易云音乐外链。

**时代荧幕（11部）**：涵盖经典影视作品，包含国产剧、动画、电影等。

**时代网络（3条）**：涵盖 Flash 小游戏、贴吧、论坛等早期互联网文化记忆。

### 3.3 展厅二：荣耀之路（Hall of Glory）

#### 3.3.1 布局结构

采用垂直时间轴布局，荣誉条目按年份从早到晚排列，左侧或居中显示时间轴线，右侧或两侧展示荣誉卡片。

#### 3.3.2 荣誉条目字段

| 字段 | 类型 | 说明 |
|------|------|------|
| year | string | 获得年份 |
| title | string | 荣誉名称 |
| description | string | 荣誉描述 |
| imageUrl | string | 证书/证明图片，Base64 或 URL |
| reflection | string | 复盘反思内容，支持较长文本 |

#### 3.3.3 默认数据

| 序号 | 荣誉名称 | 年份 |
|------|----------|------|
| 1 | 国家励志奖学金 | 待定 |
| 2 | 优秀志愿者 | 待定 |
| 3 | 数据库系统工程师证书 | 待定 |
| 4 | 网络安全证书 | 待定 |
| 5 | 英语四级证书 | 待定 |

#### 3.3.4 交互规则

- 每条荣誉默认展示年份、标题、描述和图片
- 点击"复盘反思"按钮可展开/折叠详细的反思内容，展开时使用 Framer Motion 的高度动画

## 四、数据模型

### 4.1 VintageCard 接口（展厅一通用）

```typescript
interface VintageCard {
  id: string;            // 唯一标识，使用 uuid 生成
  year: string;          // 年份，如 "2005"
  title: string;         // 标题
  description: string;   // 描述
  imageUrl: string;      // 封面图，Base64 或 URL
  musicUrl?: string;     // 音乐链接（仅 BGM 类目使用）
}
```

### 4.2 HonorItem 接口（展厅二）

```typescript
interface HonorItem {
  id: string;            // 唯一标识，使用 uuid 生成
  year: string;          // 获得年份
  title: string;         // 荣誉名称
  description: string;   // 荣誉描述
  imageUrl: string;      // 证书/证明图片，Base64 或 URL
  reflection: string;    // 复盘反思内容
}
```

### 4.3 localStorage 存储结构

| Storage Key | 数据类型 | 对应接口 | 说明 |
|-------------|----------|----------|------|
| `museum_bgms` | `VintageCard[]` | VintageCard | 时代BGM 展品列表 |
| `museum_tvs` | `VintageCard[]` | VintageCard | 时代荧幕展品列表 |
| `museum_nets` | `VintageCard[]` | VintageCard | 时代网络展品列表 |
| `museum_honors` | `HonorItem[]` | HonorItem | 荣耀之路展品列表 |

### 4.4 数据迁移逻辑

当页面加载时，系统检测各 localStorage 数组中每条数据是否包含 `imageUrl` 字段。若缺失，则自动补全为空字符串 `""`，确保接口兼容性。此逻辑在 `useEffect` 初始化阶段执行，仅触发一次。

## 五、交互逻辑与规则约束

### 5.1 管理员认证

- 使用 `useAdminGuard` 自定义 Hook 进行权限守卫
- 认证密码为 `"ling"`，硬编码于组件中
- 未认证状态下，所有编辑操作按钮隐藏
- 认证通过后，页面顶部显示管理工具栏（添加、退出登录）
- 认证状态不作持久化，页面刷新后需重新登录

### 5.2 CRUD 操作流程

**新增（Create）**
1. 管理员点击"添加"按钮
2. 弹出 `CardForm` 表单组件
3. 填写必填字段（年份、标题、描述）
4. 可选上传图片（自动转 Base64）或填入 URL
5. BGM 类目可额外填写音乐链接
6. 提交后，新数据追加到对应 localStorage 数组
7. 表单关闭，列表即时刷新

**编辑（Update）**
1. 管理员点击卡片上的编辑按钮
2. 弹出 `CardForm` 表单，预填充现有数据
3. 修改字段后提交
4. 对应 localStorage 数组中的条目被更新
5. 列表即时刷新

**删除（Delete）**
1. 管理员点击卡片上的删除按钮
2. 弹出 `ConfirmDialog` 确认对话框
3. 用户确认后，从 localStorage 数组中移除对应条目
4. 用户取消则关闭对话框，不执行任何操作

**查看（Read）**
- 所有用户（含访客）可自由浏览
- 荣耀之路的复盘反思内容默认折叠，点击展开

### 5.3 图片上传规则

- 支持格式：JPG、PNG、GIF、WEBP
- 使用 `FileReader.readAsDataURL()` 将图片转为 Base64 字符串
- Base64 字符串直接存入 localStorage 的 `imageUrl` 字段
- 单一图片文件大小建议不超过 2MB，前端不做硬性限制，但需注意 localStorage 单 Key 5MB 上限

### 5.4 音乐播放器规则

- 仅 BGM 类目支持音乐播放
- 音乐链接格式为网易云音乐 iframe 外链：`https://music.163.com/outchain/player?type=2&id={歌曲ID}&auto=0&height=66`
- 点击播放按钮后，`MiniPlayer` 组件以绝对定位弹出层形式展示 iframe 播放器
- 同一时间仅允许一个播放器处于活跃状态，打开新播放器时关闭旧的

### 5.5 约束条件

- 所有数据存储在用户浏览器本地，清除浏览器数据将导致数据丢失
- 不支持多设备同步，数据仅存在于当前浏览器
- 图片以 Base64 存储，大量高清图片可能导致 localStorage 超限
- 管理员密码 `"ling"` 为明文硬编码，仅适用于个人站点场景，不具备安全防护能力

## 七、未来迭代方向

### 7.1 V1.1 计划

- **展厅三：灵感源泉（Hall of Inspiration）**：展示影响个人成长的书籍、文章、播客等内容
- **展厅四：未来之约（Hall of Future）**：个人目标与愿景展示，支持进度跟踪
- **数据导出功能**：支持将全部展品数据导出为 JSON 文件，便于备份与迁移
- **数据导入功能**：支持从 JSON 文件导入数据，实现跨设备同步

### 7.2 V1.2 计划

- **访客互动**：支持访客在展品下留言或点赞
- **搜索与筛选**：支持按年份、关键词搜索展品
- **博物馆导览模式**：自动滚动浏览所有展厅，适合大屏展示
- **主题切换**：支持暗色模式与胶片模式的切换

### 7.3 V2.0 长期规划

- **后端接入**：将数据从 localStorage 迁移至后端数据库，支持多端同步
- **多用户支持**：每位注册用户拥有独立的博物馆空间
- **社交分享**：支持将单个展品或整个博物馆分享至社交平台
- **AI 复盘助手**：基于用户填写的荣誉与反思内容，AI 生成成长报告与建议
