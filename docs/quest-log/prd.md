# PRD -- 通关清单（Quest Log）

> 产品需求文档 | 版本 V1.0 | 2026-07-03

---

## 一、产品背景与定位

### 1.1 背景

市面上的待办事项（To-Do）工具普遍存在"功能臃肿、启动阻力大、缺乏正向激励"三大痛点。用户面对长长的任务列表容易产生逃避心理，完成任务后缺乏即时反馈，长期使用动力不足。游戏化（Gamification）是解决这一问题的有效手段，但现有游戏化 To-Do 工具多为独立 App，与日常工具生态割裂。

### 1.2 定位

通关清单是妙妙工具箱「行」维度的核心工具，定位为**RPG 风格的游戏化待办清单**。它将每一个待办事项转化为"冒险任务"，通过 XP 经验值系统、智能拆解弹窗、粒子爆炸完成动画和金币音效，把"做任务"变成"打怪升级"。

### 1.3 产品价值主张

- **游戏化激励**：将枯燥的 To-Do 转化为 RPG 冒险，通过 Lv 等级、XP 进度条、完成特效提供即时爽感
- **智能拆解**：对于感到困难的任务，提供三种风险收益选项，降低启动阻力
- **极简克制**：不追求功能大而全，只做"添加-选择难度-完成"的核心闭环
- **隐私保护**：所有数据存储在本地 localStorage，无需注册登录

### 2.1 用户画像

| 画像 | 特征 | 痛点 |

| 拖延症青年 | 20-30 岁，自驱力不足，容易被任务量吓退 | 需要降低启动门槛，需要即时正向反馈 |

| 效率工具爱好者 | 22-35 岁，使用过多种 To-Do 工具 | 厌倦传统 To-Do 的枯燥，追求新鲜交互体验 |

| 场景 | 触发时机 | 用户行为 | 预期结果 |
|------|----------|----------|----------|
| 每日任务规划 | 早晨/工作开始前 | 输入 3-5 个任务，选择难度 | 明确当日优先级，降低焦虑 |

### 3.2 模块详情

#### 3.2.1 任务管理

**添加任务流程：**
1. 用户在底部输入框输入任务描述
2. 按回车或点击"添加"按钮
3. 弹出智能拆解弹窗，选择难度模式
4. 任务添加到列表顶部

**完成/删除交互：**
- 完成按钮：绿色渐变，hover 放大 + 发光
- 编辑按钮（铅笔图标）：hover 显形，点击打开编辑模态框
- 删除按钮（垃圾桶图标）：hover 变红，点击需管理员验证

**任务项状态：**
- 进行中：正常显示，含难度标签 + 文案 + 操作按钮
- 5分钟挑战中：额外显示倒计时（MM:SS 格式，黄色高亮）
- 已完成：透明度 0.5

#### 3.2.2 智能拆解弹窗

**触发条件**：用户输入新任务并提交时自动弹出。

**三种选项：**

| 选项 | 难度 | XP | 机制 | 文案 |
|------|------|-----|------|------|
| 先做 5 分钟 | easy | +5 | 倒计时 300 秒自动完成，标题追加"（5分钟挑战）" | 倒计时自动完成 |
| 只要 60 分 | normal | +60 | 标为及格线，标题追加"（及格线）" | 标为及格线，低难度 |
| 直接挑战 | hard | +100 | 满分完成，原样添加 | 满分完成，原样添加 |

**弹窗 UI：**
- 深色背景遮罩层（backdrop-filter 模糊）
- 卡片式弹窗（spring 动画弹出）
- 任务标题黄色高亮显示
- 三行选项，每行含图标 + 标签 + 描述 + XP 标签
- 各选项 hover 时对应颜色边框发光
- 关闭按钮 + 点击遮罩关闭

#### 3.2.3 XP 经验值系统

**等级计算：**
- 每 100 XP 升一级
- 等级从 Lv.1 开始
- 公式：`level = floor(xp / 100) + 1`

**XP 进度条：**
- 霓虹渐变（粉 -> 黄 -> 绿）填充
- 发光外阴影
- Spring 动画平滑过渡
- 显示文字：`Lv.X` 标签 + `当前经验/100 XP`

**XP 持久化：**
- localStorage key: `quest_log_xp`
- 存储为字符串数字

#### 3.2.4 视觉反馈

**粒子爆炸（完成动画）：**
- 14 个彩色粒子（霓虹粉/电光蓝/柠檬黄/绿/紫）
- 随机方向飞出（dx: -90~90, dy: -120~-40）
- 0.6s easeOut 动画：opacity 1->0, scale 1->0.2
- 粒子结束后任务项从列表中移除

**金币音效（Web Audio）：**
- 两段升频方波：B5(988Hz) -> E6(1319Hz)
- 间隔 80ms，各持续 180ms
- 音量 0.18，指数衰减

**任务退出动画：**
- 滑出 + 缩小（x: 60, scale: 0.8, opacity: 0）
- 0.35s 过渡

#### 3.2.5 编辑任务模态框

**打开方式：** 点击任务项编辑按钮（铅笔图标）

**表单内容：**
- 任务描述输入框（自动聚焦）
- 难度等级选择：三个按钮（easy/normal/hard），带图标 + 标签 + XP 值
- 选中态：边框发光 + 对应颜色背景
- 取消/保存按钮

**保存逻辑：**
- 需要管理员验证（`verifyAdmin`）
- 保存后模态框关闭，列表更新

#### 3.2.6 管理员保护

**保护范围：** 删除任务、编辑任务

**验证方式：**
- 通过 `useAdminGuard` hook
- 未验证时弹出 `AdminGuardUI` 验证界面
- 验证通过后执行操作

**管理员入口：**
- 右下角浮动圆形按钮
- 未登录：锁图标，半透明白色
- 已登录：齿轮图标，半透明绿色
- 点击触发 `verifyAdmin`

## 四、数据模型

### 4.1 任务数据（localStorage key: `quest_log_data`）

```typescript
interface Quest {
  id: string;          // UUID 或时间戳 ID
  text: string;        // 任务描述
  difficulty: "easy" | "normal" | "hard";
  completed: boolean;  // 是否完成
  countdown?: number;  // "5分钟挑战"倒计时剩余秒数（仅 easy 模式）
}
```

### 4.2 XP 数据（localStorage key: `quest_log_xp`）

```typescript
// 存储为字符串数字，例如 "350"
type XP = string;
```

### 4.3 预设数据

```typescript
const DEFAULT_QUESTS: Quest[] = [
  { id: "q1", text: "读完 10 页书", difficulty: "easy", completed: false },
  { id: "q2", text: "搞定周报初稿", difficulty: "hard", completed: false },
  { id: "q3", text: "喝一杯水", difficulty: "easy", completed: true },
];
```

## 五、交互逻辑与规则约束

### 5.1 任务生命周期

| 状态 | 触发 | 操作 |
|------|------|------|
| 新建 | 输入框提交 | 弹出智能拆解弹窗 |
| 选择难度 | 弹窗中选择 | 任务添加到列表，easy 模式启动倒计时 |
| 进行中 | 列表展示 | 可编辑、可完成、可删除 |
| 倒计时中 | easy 模式 | 每秒递减，归零自动完成 |
| 完成 | 点击完成/倒计时归零 | 粒子爆炸 + 金币音效 + XP 奖励，任务从列表移除 |
| 删除 | 点击删除 | 管理员验证后移除 |

### 5.2 规则约束

| 规则 | 说明 |
|------|------|
| 空任务拦截 | 输入为空时添加按钮 disabled |
| 已完成任务 | 不显示"完成"和"编辑"按钮，只显示删除按钮 |
| 倒计时任务 | 任务存在 `countdown` 字段且 > 0 时，显示黄色倒计时 |
| 难度标签 | 显示对应中文文案，不可点击修改（需通过编辑弹窗） |
| XP 不减少 | 删除任务不扣减 XP |
| 重复完成 | 已完成的预设任务不会重复计入 XP |

### 5.3 5分钟挑战倒计时

| 规则 | 说明 |
|------|------|
| 初始值 | 300 秒（5 分钟） |
| 更新频率 | 每秒递减 |
| 显示格式 | MM:SS（例如 04:59） |
| 归零行为 | 自动触发完成逻辑 |
| 跨页面 | 刷新页面后倒计时状态从 localStorage 恢复 |

## 六、验收标准

### 6.1 功能验收

| 验收项 | 标准 |
|--------|------|
| 添加任务 | 输入文本后提交，弹出智能拆解弹窗 |
| 智能拆解 | 三种选项可点击，选择后任务正确添加到列表 |
| 5分钟挑战 | 倒计时正确显示，每秒递减，归零自动完成 |
| 完成任务 | 粒子爆炸动画正常，金币音效播放，XP 正确增加 |
| XP 系统 | 等级正确计算，进度条动画平滑 |
| 删除任务 | 需管理员验证，验证后任务移除 |
| 编辑任务 | 模态框可修改描述和难度，保存后列表更新 |
| 数据持久化 | 刷新页面后任务列表和 XP 数据正确恢复 |

### 6.2 兼容性验收

| 验收项 | 标准 |
|--------|------|
| 浏览器 | Chrome 90+、Edge 90+、Firefox 90+、Safari 15+ |
| 移动端 | iOS Safari 15+、Android Chrome 90+ |
| Web Audio | 不支持 Web Audio 时音效静默失败 |
| crypto.randomUUID | 不支持时降级为时间戳 + 随机字符串 |

### 6.3 性能验收

| 验收项 | 标准 |
|--------|------|
| 粒子动画 | 14 个粒子同时动画不卡顿 |
| 倒计时 | 多个倒计时任务同时运行正常 |
| 内存 | 添加/删除大量任务无明显内存泄漏 |

## 七、未来迭代方向

### 7.1 V1.1 规划

- **任务分类**：支持自定义标签/分类（工作/生活/学习）
- **成就系统**：里程碑成就徽章（连续打卡/累计完成 100 个任务等）
- **每日任务刷新**：每日自动重置的重复任务模板
- **任务排序**：拖拽排序、按难度/时间排序

### 7.2 V1.2 规划

- **数据统计**：完成率趋势图、难度分布饼图、周/月完成热力图
- **导出功能**：任务数据导出为 JSON/CSV
- **音效自定义**：多种完成音效可选（金币/升级/魔法等）
- **主题切换**：更多 RPG 主题皮肤（冰霜/火焰/暗影）

### 7.3 V2.0 规划

- **多人协作**：共享任务列表，团队冒险
- **Boss 战**：大型任务拆解为多个子任务，完成后触发 Boss 击败动画
- **装备系统**：完成特定类型任务获得虚拟装备，影响 XP 倍率
- **云端同步**：可选账号登录，数据跨设备同步
