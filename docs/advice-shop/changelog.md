# 解忧杂货铺 (The Advice Shop) -- CHANGELOG

## V1.0 (2026-07-03) -- 首次发布

### 新增功能

- 信纸风格问题输入界面：米白底色横线纹理信纸，含温馨开场白和占位提示
- 温馨开场白：三句随机切换（"夜安，旅人。" / "嘿，我在听。" / "别急，慢慢说。"），基于输入长度索引稳定切换
- 投递按钮：全宽渐变金按钮，支持 Ctrl+Enter (Windows) / Cmd+Enter (Mac) 快捷键提交
- 输入校验：问题文本非空才可提交，加载中时按钮禁用并显示"正在写信..."
- AI 回信：接入阿里云百炼 DeepSeek 模型，通过 OpenAI 兼容 API 格式调用，固定角色为"晴美"
- 角色系统提示词：精心设计的晴美人设，包含核心风格（松弛感、自然接话、幽默、简短）和绝对禁止项（禁写信词汇、禁署名、禁动作描写、禁 AI 口吻、禁自曝身份）
- 内置两个示例对话：饿了不知道吃什么 / 工作太累想辞职，作为 LLM 提示词中的 few-shot 示例
- 伪流式打字机效果：完整获取回信后，前端逐字打印，每字间隔 30ms，模拟真实写信体验
- 回信卡片：金色边框卡片（#ecd9b8），含"已送达"旋转印章（右上角，8 度旋转）、"亲爱的旅人："称呼、角色署名、正文、"—— 晴美 · 解忧杂货店"签名
- 加载动画：钢笔图标摇摆动画（rotate: [-3, 3, -3]，1.2 秒循环）+ "杂货店老板正在回信..." 文案
- 请求取消：使用 AbortController 管理请求生命周期，新请求提交时自动取消上一次未完成请求
- 降级处理：环境变量未配置时，600ms 后展示默认回信"网有点卡，稍等一会儿呗。"
- 错误处理：网络错误时展示"信还在路上...\n\n（调试信息：具体错误信息）"；空响应时抛出配置检查提示
- 再写一封：一键清空回信和输入框，回到初始信纸状态，可开启新一轮对话
- 烛光浮动动画：蜡烛图标（🕯️）持续上下浮动，3 秒循环，带暖光滤镜效果
- 标题渐入动画：标题"解忧杂货铺"和副标题"总有一句话，能解开你的心结。"依次淡入，延迟 0.12s
- 信纸区域渐入动画：信纸输入区从下方淡入，延迟 0.2s
- 回信升起动画：回信卡片从下方 40px 升入，0.7 秒弹性缓出；退场时淡出并下移 20px
- 返回导航：顶栏"回到妙妙工具箱"链接，路由至 `/mickey`

### 技术实现

- 基于 React 18 + TypeScript 构建，使用函数组件 + Hooks 模式
- 动画系统采用 framer-motion，使用 AnimatePresence 管理回信卡片的入场/退场动画
- 伪流式打字机通过 `for` 循环 + `await setTimeout` + `setReply` 逐步更新实现
- 请求管理使用 `useRef` 持有 `AbortController` 实例，每次新请求前调用 `abort()` 取消旧请求
- 环境变量通过 `import.meta.env.VITE_API_KEY` / `VITE_BASE_URL` / `VITE_MODEL` 读取
- 开场白使用 `useMemo` 缓存，基于 `question.length % 3` 索引切换，避免不必要的重新计算
- 响应式设计：最大内容宽度 680px 居中，移动端（<=520px）信纸和回信卡片内边距缩减
- 全组件采用内联样式 + CSS-in-JS（组件内 `<style>` 标签），无外部 CSS 依赖
- 字体栈：Noto Serif SC（衬线，标题和回信正文） + Noto Sans SC（无衬线，UI 文字） + system-ui（回退）
- 背景实现：径向渐变暖光 + SVG 噪点纹理（fractalNoise 滤镜），营造纸张质感
- 按钮渐变：linear-gradient(135deg, #c8924a, #b07832)，金色渐变 + 投影阴影
- 信纸横线：repeating-linear-gradient 实现 32px 间距横线纹理
- 回信印章：transform: rotate(8deg) 实现轻微旋转，模拟真实印章效果
