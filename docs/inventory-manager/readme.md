# 物资管家 (Inventory Prophet) -- README

## 产品简介

物资管家是一款轻量级个人/家庭库存管理工具，内置于妙妙工具箱（Mickey Toolbox）"管"维度模块。它帮助用户追踪食品、日用品、药品等物资的库存状态，通过到期日预警和智能消耗建议，将资源浪费控制在萌芽状态。

**核心理念**: 把每一件物品用在它最好的时候。

## 功能列表

| 功能模块 | 功能描述 |
|----------|----------|
| 入库登记 | 录入物品名称、数量、单位、到期日、存放位置，支持连续入库 |
| 状态仪表盘 | 三态胶囊（已过期/临期30天/库存充足）实时统计，点击可筛选 |
| 库存列表 | 表格展示全部物品，彩色状态标识，相对时间文案 |
| 编辑弹窗 | 模态弹窗编辑物品全部字段，预填充当前值 |
| 删除动画 | 300ms 淡出动画，操作前需管理员验证 |
| 智慧规划 | 自动识别最早过期物品，提供差异化消耗建议 |
| 权限控制 | 编辑/删除操作需管理员密码验证，防止误操作 |
| 数据持久化 | 基于 localStorage 自动存储，刷新不丢失 |

## 本地存储说明

### 存储键名

`inventory_items`

### 数据结构

```typescript
interface InventoryItem {
  id: string;          // UUID v4 或 fallback 格式
  name: string;        // 物品名称
  count: number;       // 数量，>= 1
  unit: Unit;          // 单位（瓶/盒/袋/罐/箱/个/支）
  expiryDate: string;  // 到期日，YYYY-MM-DD
  location: Location;  // 存放位置（冰箱/浴室/储物间/厨房/客厅/卧室）
}
```

### 存储行为

- **写入时机**: 每次库存数据变更时自动同步（React useEffect 监听）
- **读取时机**: 页面初始化时从 localStorage 加载
- **容错处理**: 读取失败（如 JSON 解析错误）返回空数组；写入失败（如隐私模式）静默忽略
- **数据生命周期**: 与浏览器缓存一致，清除浏览器数据后数据丢失

### 数据安全

- 数据完全存储在用户本地浏览器中，不上传至任何服务器
- 编辑和删除操作需要管理员密码验证，防止误操作
- 无账户体系，无隐私泄露风险

## 设计理念

### 极简入库流程

入库是最高频操作，设计上追求极致效率。表单字段精简到 5 个核心字段，入库后仅清空名称字段，保留数量和位置，方便用户连续录入同类物品。整个入库流程目标控制在 30 秒内完成。

### 三态视觉模型

采用红/橙/绿三色体系，直观传达库存状态。状态判定基于距到期日天数，临期阈值为 30 天。每个状态胶囊既是统计指示器，也是筛选开关，一键切换视图。

### 无感持久化

用户无需关心数据存储，所有操作自动同步到 localStorage。页面刷新后数据完整保留，没有任何加载或同步等待。即使写入失败（如隐私模式），也不影响正常使用，只是数据不持久化。

### 管理员权限模型

物资管家的权限设计遵循"宽松读取、严格写入"原则。查看库存完全自由，但编辑和删除操作需要管理员密码验证。这一设计在保障数据安全的同时，保持了日常使用（特别是入库登记）的流畅体验。

### 鼠尾草绿视觉语言

主色调采用鼠尾草绿 (#5d8a6a)，传达自然、健康、可持续的品牌感受，与"反浪费"的产品定位高度一致。整体布局采用 SaaS 风格的白色卡片 + 浅灰背景，紧凑而专业。
