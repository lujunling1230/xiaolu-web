## 1. 产品背景与定位

### 1.1 产品背景

在家庭日常生活中，物资管理长期依赖个人记忆和零散便签。冰箱里的食材、储物间的日用品、药箱里的药品，常常因为缺乏系统性记录而发生过期浪费。据统计，一个普通家庭每年因食品和日用品过期造成的浪费可占总支出的 5%-8%。用户并非不愿管理，而是缺少一个足够轻量、足够直观、没有任何使用门槛的库存管理工具。

物资管家 (Inventory Prophet) 正是为解决这一痛点而生。它不追求企业级 WMS（仓库管理系统）的复杂度，而是聚焦于"个人/家庭库存管理"这一核心场景，提供入库登记、过期预警、智能消耗建议三大核心能力，将资源浪费控制在萌芽状态。

### 1.2 产品定位

物资管家是「妙妙工具箱」中"管"维度的代表性工具，定位于**资源管理与反浪费**。在全站 9 维度体系中，"管"维度关注的是对生活资源的系统化管理能力，其核心理念是"把每一件物品用在它最好的时候"。

| 维度 | 产品定位 | 关键指标 |
|------|----------|----------|
| 管 (Manage) | 个人/家庭库存管理 SaaS | 过期物品数量、临期提醒覆盖率 |
| 风格 | 紧凑型 SaaS 工具 | 白色卡片 + 浅灰背景 + 鼠尾草绿主色调 |
| 用户心智 | 省心管家 | "有人帮我盯着，不会浪费" |

### 1.3 产品目标

- **降低浪费**: 通过临期提醒和智能规划，帮助用户优先消耗即将过期的物品
- **提升效率**: 入库登记流程控制在 30 秒内完成，无需登录或注册
- **数据持久化**: 基于 localStorage 的本地存储方案，数据完全由用户掌控，无隐私泄露风险

## 2. 用户画像与核心场景

### 2.1 用户画像

| 画像标签 | 描述 | 典型行为 |
|----------|------|----------|
| 独居青年 | 25-35 岁，独居，经常批量采购食材 | 周末大采购后分批入库，关注临期牛奶和蔬菜 |
| 家庭主理人 | 30-50 岁，负责家庭物资采购 | 管理冰箱、储物间、浴室多位置库存，定期清理过期物品 |
| 极简生活者 | 25-40 岁，追求"少而精"的生活方式 | 严格控制库存数量，每件物品入库即规划消耗时间 |
| 合租室友 | 22-30 岁，合租共享冰箱和储物空间 | 多人共用，需要明确物品归属和到期时间 |

### 2.2 核心场景

**场景一：周末采购后入库**

> 用户周末去超市采购了一批食材和日用品，回到家后打开物资管家，依次录入物品名称、数量、到期日、存放位置。系统自动计算到期天数，并在赏味期限面板中提示最早到期的物品。

**场景二：日常消耗决策**

> 打开冰箱不知道先吃什么。用户查看物资管家仪表盘，发现"临期 30 天内"标签下有 3 件物品，赏味期限面板推荐了"最早过期物品"，用户优先消耗该物品。

**场景三：定期清理过期物品**

> 月初大扫除，用户进入物资管家，点击"已过期"胶囊筛选出所有过期物品，逐项确认后删除。删除操作需要管理员验证，防止误删。

**场景四：库存盘点与调整**

> 用户发现某物品数量有误，点击表格中的"编辑"按钮，在弹出的编辑弹窗中修改数量或到期日，保存后自动更新。

### 3.2 模块详情

#### 3.2.1 入库登记模块

**功能描述**: 位于页面左侧上方，提供物品入库登记表单。

**表单字段**:

| 字段名 | 类型 | 必填 | 默认值 | 校验规则 |
|--------|------|------|--------|----------|
| 物品名称 | text | 是 | 空 | 去除首尾空格后非空 |
| 数量 | number | 是 | 1 | 最小值 1 |
| 单位 | select | 否 | 瓶 | 7 个预设选项 |
| 到期日 | date | 是 | 空 | 必选日期 |
| 存放位置 | select | 否 | 冰箱 | 6 个预设选项 |

**单位选项**: 瓶、盒、袋、罐、箱、个、支

**存放位置选项**: 冰箱、浴室、储物间、厨房、客厅、卧室

**交互细节**:
- "确认入库"按钮在表单校验通过前保持禁用状态（灰色 + 50% 透明度）
- 入库成功后仅清空物品名称字段，保留数量、单位、位置，便于连续入库同类物品
- 新入库物品出现在列表顶部（倒序插入）

#### 3.2.2 状态仪表盘模块

**功能描述**: 位于页面右侧顶部，以三个胶囊按钮展示库存状态统计。

**三种状态判定规则**:

| 状态 | 判定条件 | 颜色 | 胶囊样式 |
|------|----------|------|----------|
| 已过期 | 到期日 < 今天 | 红色 #E53935 | 红底红字 |
| 临期 | 0 <= 距到期天数 <= 30 | 琥珀色 #FF9800 | 琥珀底琥珀字 |
| 库存充足 | 距到期天数 > 30 | 翡翠绿 #81C784 | 翡翠底翡翠字 |

**交互逻辑**:
- 点击任一胶囊，表格列表仅显示该状态的物品
- 再次点击同一胶囊，取消筛选，显示全部物品
- 筛选激活时，胶囊呈现高亮激活态（彩色背景），非激活态为白色背景
- 任意筛选激活时，右侧出现"显示全部"文字按钮，点击后恢复全部显示

#### 3.2.3 库存列表模块

**功能描述**: 位于页面右侧主体区域，以表格形式展示库存物品。

**表格列定义**:

| 列名 | 数据来源 | 显示格式 |
|------|----------|----------|
| 物品 | name | 文字 + 状态标签（过期时显示红色"已过期"徽章） |
| 数量 | count + unit | "2 瓶"格式 |
| 到期日 | expiryDate | 日期 + 相对时间提示（"还剩 15 天"） |
| 存放位置 | location | 文字 |
| 操作 | - | "编辑" + "删除"按钮 |

**视觉标识**:
- 每行左侧有 4px 宽的彩色竖条，颜色与状态对应
- 到期日文字颜色与状态对应
- 过期物品名称显示为红色

**空状态**:
- 全局无物品时: "还没有任何物品，去左侧入库吧。"
- 筛选后无物品时: "当前筛选下没有物品。"

#### 3.2.4 编辑弹窗模块

**功能描述**: 点击表格中的"编辑"按钮，弹出模态编辑弹窗。

**弹窗内容**:
- 标题: "编辑物品"
- 表单字段: 与入库登记完全一致，预填充当前值
- 按钮: "取消"（关闭弹窗）+ "保存修改"（提交更新）

**交互规则**:
- 点击遮罩层关闭弹窗（不保存）
- 点击弹窗内部阻止事件冒泡（不关闭）
- 保存前需通过管理员验证（调用 useAdminGuard.verifyAdmin）
- 保存成功后自动关闭弹窗，列表即时更新

#### 3.2.5 删除功能模块

**功能描述**: 点击表格中的"删除"按钮或赏味期限面板中的"立即消耗"按钮，删除指定物品。

**动画效果**:
- 删除前先触发 300ms 淡出动画（opacity 过渡至 0）
- 动画结束后从数据和 DOM 中移除

**权限控制**:
- 删除操作前需通过管理员验证
- 若未登录管理员，弹出密码输入框，验证通过后执行删除

#### 3.2.6 智慧规划模块

**功能描述**: 位于页面左侧下方，赏味期限面板，基于库存数据提供智能消耗建议。

**核心逻辑**:
- 按到期日升序排序，取最早到期的物品
- 计算该物品距到期的天数
- 根据天数分支展示不同文案和操作

**文案分支**:

| 天数范围 | 主文案 | 按钮文案 | 按钮颜色 |
|----------|--------|----------|----------|
| <= 30 天 | "赏味期限将至，宜趁鲜启用。" | "立即消耗" | 暖橙 #D46B4D |
| > 30 天 | "余量丰盈，且容它静候时光。" | "留着以后" | 鼠尾草绿 #557C55 |

**空状态**: "暂无物品。先在上方入库，我来帮你规划。"

## 4. 数据模型

### 4.1 核心数据结构

```typescript
interface InventoryItem {
  id: string;          // 唯一标识，UUID v4 或 fallback 格式
  name: string;        // 物品名称
  count: number;       // 数量，整数 >= 1
  unit: Unit;          // 单位，枚举值
  expiryDate: string;  // 到期日，YYYY-MM-DD 格式
  location: Location;  // 存放位置，枚举值
}

type Unit = "瓶" | "盒" | "袋" | "罐" | "箱" | "个" | "支";
type Location = "冰箱" | "浴室" | "储物间" | "厨房" | "客厅" | "卧室";
```

### 4.2 派生状态

```typescript
type Status = "expired" | "expiring" | "sufficient";

// 状态判定函数
function toStatus(days: number): Status {
  if (days < 0) return "expired";      // 已过期
  if (days <= 30) return "expiring";   // 临期 30 天内
  return "sufficient";                 // 库存充足
}
```

### 4.3 存储方案

| 属性 | 说明 |
|------|------|
| 存储方式 | 浏览器 localStorage |
| 存储键名 | `inventory_items` |
| 数据格式 | JSON 序列化的 InventoryItem[] |
| 同步时机 | 每次 items 状态变更时自动同步（useEffect 监听） |
| 容错处理 | 读取失败返回空数组；写入失败静默忽略（隐私模式等） |

## 5. 交互逻辑与规则约束

### 5.1 状态判定规则

- 到期日比较基于当天零点（`setHours(0,0,0,0)`），避免跨天比较偏差
- 到期日当天（0 天）判定为"临期"，文案显示"今天到期"
- 负数天数表示已过期，文案显示"已过期 N 天"

### 5.2 权限控制

| 操作 | 是否需要管理员权限 | 说明 |
|------|-------------------|------|
| 入库登记 | 否 | 任何用户可自由添加物品 |
| 编辑物品 | 是 | 需通过密码验证 |
| 删除物品 | 是 | 需通过密码验证 |
| 查看库存 | 否 | 所有数据公开可读 |

管理员验证通过 `useAdminGuard` Hook 统一管理，验证成功后在会话期间保持管理员状态（退出登录前持续有效）。

### 5.3 路由配置

| 路由 | 页面 | 说明 |
|------|------|------|
| `/toolbox/inventory` | 物资管家 | 主路由 |
| `/toolbox/supplies` | 物资管家 | 别名路由，妙妙工具箱入口专用 |

返回导航: 点击顶栏"妙妙工具箱"链接返回 `/mickey`，支持透传 `?from=full` 参数。

### 5.4 响应式布局

- 桌面端（>=1024px）: 左右双栏布局，左侧 340px 固定侧边栏 + 右侧弹性主体
- 移动端（<1024px）: 单栏垂直布局，左侧面板置于上方
- 左侧面板在桌面端使用 sticky 定位，滚动时保持可见

### 5.5 边界条件

- 物品名称需去除首尾空格后校验非空
- 数量输入框最小值为 1，输入非法值时自动回退为 1
- 到期日字段使用 HTML5 date input，浏览器原生日期选择器
- ID 生成优先使用 `crypto.randomUUID()`，兼容回退为 `时间戳-随机字符串`

## 7. 未来迭代方向

### 7.1 V1.1 短期迭代

- **批量入库**: 支持 CSV 或扫一扫批量导入物品
- **搜索功能**: 在库存列表中增加关键词搜索
- **排序切换**: 支持按到期日、名称、入库时间排序
- **导出功能**: 导出库存清单为 CSV 或 PDF

### 7.2 V1.2 中期规划

- **分类标签**: 物品支持自定义分类（食品/日用品/药品等）
- **消耗记录**: 每次消耗记录消耗量，追踪使用趋势
- **消耗预测**: 基于历史消耗数据预测下次采购时间
- **库存预警推送**: 支持浏览器通知，到期前主动提醒
- **共享库存**: 支持多人协作管理（家庭群组）

### 7.3 V2.0 远期愿景

- **AI 采购建议**: 结合库存和消耗趋势，智能生成采购清单
- **食谱推荐**: 基于临期食材推荐菜谱
- **云端同步**: 多设备数据同步（需用户账户体系）
- **语音入库**: 语音输入物品信息，自动识别并入库
