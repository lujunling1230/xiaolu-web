# 解忧杂货铺 (The Advice Shop) -- README

## 产品简介

解忧杂货铺是一款治愈系 AI 情感问答空间，内置于妙妙工具箱（Mickey Toolbox）"术"维度模块。它以信纸风格的界面呈现，用户输入问题后，由固定角色"晴美"（解忧杂货店掌柜）通过 AI 生成回信，以伪流式打字机效果逐字展示，营造出"正在写信"的真实感。

**核心理念**: 总有一句话，能解开你的心结。

灵感来源于东野圭吾同名小说《解忧杂货店》。

## 功能列表

| 功能模块 | 功能描述 |
|----------|----------|
| 信纸界面 | 米白底 + 金色边框 + 横线纹理信纸，温馨开场白，Ctrl+Enter 快捷键 |
| AI 回信 | 接入阿里云百炼 DeepSeek 模型，固定角色"晴美"，伪流式打字机逐字打印 |
| 回信卡片 | 金色边框卡片，含"已送达"旋转印章、角色署名、签名 |
| 加载动画 | 钢笔摇摆动画 + "杂货店老板正在回信..." 提示 |
| 请求管理 | 新请求自动取消上一次未完成请求，避免回信混乱 |
| 降级处理 | 未配置 API 时展示默认回复，网络错误时展示调试信息 |
| 再写一封 | 一键清空回信和输入框，回到初始状态 |
| 视觉动效 | 烛光浮动、标题渐入、回信升起、framer-motion 流畅动画 |

## 环境变量配置

在项目根目录 `.env` 文件中配置：

```bash
VITE_API_KEY=sk-xxxxxxxxxxxxxxxx          # 阿里云百炼 API Key
VITE_BASE_URL=https://your-api-endpoint   # API 端点 URL（OpenAI 兼容格式）
VITE_MODEL=deepseek-r1                    # 模型名称
```

若未配置环境变量，解忧杂货铺将展示默认降级回信，不会发送 API 请求。

## 本地存储说明

**解忧杂货铺不进行任何数据持久化。** 所有对话内容仅存在于当前浏览器会话中，页面刷新或关闭后数据完全消失。这一设计出于隐私保护考量 -- 用户倾诉的内容不留下任何痕迹。

## 设计理念

### 信纸美学

解忧杂货铺的视觉设计围绕"信纸"这一核心意象展开。米白色背景（#faf6ee）模拟纸张质感，金色边框（#ece2cf）传达温暖与精致，横线纹理（repeating-linear-gradient）还原真实信纸的书写体验。衬线字体（Noto Serif SC）的选择进一步强化了"书信"的仪式感。

### 角色人格化

与市面上大多数 AI 对话产品不同，解忧杂货铺没有采用"助手"定位，而是创造了一个具象的角色 -- 晴美。她是解忧杂货店的掌柜，一个普通但有趣的女生，用松弛、幽默、接地气的方式回应用户。系统提示词经过精心设计，明确了核心风格和绝对禁止项，确保 AI 输出始终符合角色人格。

### 伪流式打字机

采用非流式 API 请求 + 前端逐字打印的混合策略。完整获取回信后再逐字展示，避免了 SSE 流式传输可能出现的断连残缺问题，同时保留了"正在写信"的真实感和期待感。每字 30ms 的间隔经过调优，既不会太快失去仪式感，也不会太慢让用户失去耐心。

### 无痕设计

不存储任何对话数据，是解忧杂货铺最核心的设计决策之一。用户的情感倾诉往往是私密的，不留痕迹的设计让用户能够放心表达。同时，这也降低了技术复杂度，避免了数据管理和隐私合规的负担。

### 暖色调视觉语言

整体配色围绕暖金色系展开：米白底（#faf6ee）、金色边框（#ece2cf）、渐变金按钮（#c8924a -> #b07832）、暖色文字（#4a4036）。背景使用径向渐变制造柔和的暖光效果，配合 SVG 噪点纹理增加纸张质感。蜡烛图标（filter: drop-shadow 暖光）作为页面核心视觉锚点，强化"温暖小店"的空间感。

## 角色设定

### 晴美 (Harumi)

| 维度 | 设定 |
|------|------|
| 身份 | 解忧杂货店的看店掌柜 |
| 性格 | 普通但有趣，松弛感，幽默风趣 |
| 语言风格 | 口语化，短小精悍，允许语气词 |
| 禁止行为 | 不提及写信流程词汇、不署名、不动作描写、不 AI 口吻、不自曝身份 |
